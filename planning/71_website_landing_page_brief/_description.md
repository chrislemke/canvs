# Website / Landing Page Brief

## Overview

A Website/Landing Page Brief is a planning document that defines the objectives, content, design, and technical requirements for a web page or site. It serves as a blueprint for creating effective web experiences that communicate value, convert visitors, and support business goals. Whether for a marketing landing page, product page, or full website redesign, a clear brief aligns stakeholders and guides the design and development process.

## Purpose

- **Align stakeholders**: Ensure shared vision for the page
- **Define requirements**: Clarify content and functionality needs
- **Guide design**: Provide direction for visual design
- **Optimize conversion**: Focus on goals and user actions
- **Reduce iterations**: Prevent scope creep and rework
- **Document decisions**: Create reference for future updates
- **Enable measurement**: Define success metrics

## When to Create

- **New Pages**: Before creating any new web page
- **Redesigns**: When updating existing pages
- **Product Launches**: For launch marketing pages
- **Campaigns**: For campaign-specific landing pages
- **Website Overhauls**: For larger site projects
- **A/B Tests**: When testing new page variations

## Landing Page Structure

### Above the Fold
- **Headline**: Clear value proposition
- **Subheadline**: Supporting benefit
- **Hero image/video**: Visual representation
- **Primary CTA**: Clear action button

### Body
- **Benefits section**: Key value points
- **Features**: Product capabilities
- **Social proof**: Testimonials, logos
- **How it works**: Process explanation

### Below the Fold
- **Detailed information**: Deeper content
- **FAQ**: Common questions
- **Secondary CTA**: Repeated action
- **Trust elements**: Guarantees, security

### Footer
- **Navigation**: Key links
- **Contact**: Support options
- **Legal**: Privacy, terms

## Page Types

| Type | Purpose | Key Elements |
|------|---------|--------------|
| Homepage | Overview and navigation | Value prop, key features, CTAs |
| Product Page | Product details | Features, pricing, demo |
| Landing Page | Campaign conversion | Single focus, form, CTA |
| Pricing Page | Plan comparison | Tiers, features, FAQ |

## SEO Considerations

### On-Page SEO
- Target keyword in title and H1
- Meta description (150-160 chars)
- URL structure
- Header hierarchy (H1, H2, H3)
- Alt text for images
- Internal linking

### Technical SEO
- Page load speed (<3 seconds)
- Mobile responsiveness
- Schema markup
- Canonical URLs
- Open Graph tags

## Best Practices

1. **Start with Goals**: Define what success looks like.

2. **Know Your Audience**: Write for specific personas.

3. **Clear Value Proposition**: Communicate benefit in 5 seconds.

4. **One Primary CTA**: Focus on the main action.

5. **Mobile First**: Design for mobile, enhance for desktop.

6. **Social Proof**: Include testimonials and logos.

7. **Fast Load Times**: Optimize for speed.

8. **Test and Iterate**: A/B test key elements.

## Common Pitfalls

- **Too Many CTAs**: Confusing visitors with options
- **Weak Headlines**: Vague or feature-focused
- **No Social Proof**: Missing credibility elements
- **Slow Load Times**: Pages that frustrate users
- **Poor Mobile Experience**: Not optimized for phones
- **Missing SEO**: Not optimized for search
- **Jargon**: Language customers don't understand
- **No Clear Path**: Unclear next steps
